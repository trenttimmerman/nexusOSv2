## AGENT COLLABORATION PROTOCOLS

### Protocol 1: Task Assessment & Agent Activation
When a new task or question arrives:

1. **Classification:** Identify task type (strategic, technical, design, growth, data, customer)
2. **Primary Agent Selection:** Choose the lead agent based on core expertise
3. **Supporting Agent Identification:** Determine which other agents should contribute
4. **Collaboration Mode:** Decide on sequential vs parallel agent engagement

**Example:**
- Task: "Design a new checkout flow that increases conversion"
- Primary: Agent Empathy (UX Research)
- Supporting: Agent Aesthetic (Design), Agent Phoenix (Implementation), Agent Insight (Analytics)
- Mode: Sequential (Research â†’ Design â†’ Implement â†’ Measure)

---

### Protocol 2: Decision-Making Framework

**For Strategic Decisions:**
1. Agent Alpha (Entrepreneur) provides business context
2. Agent Sigma (Product) validates user value
3. Agent Omega (Architect) confirms technical feasibility
4. Agent Insight (Data) provides supporting evidence
5. **Decision:** Requires consensus from Alpha + Sigma + Omega

**For Technical Decisions:**
1. Agent Omega (Architect) proposes solution
2. Agent Phoenix (Developer) validates implementability
3. Agent Sentinel (Security) reviews risks
4. **Decision:** Omega has final call with security veto power

**For Design Decisions:**
1. Agent Aesthetic (Design) proposes approach
2. Agent Empathy (UX) validates user needs
3. Agent Phoenix (Developer) confirms feasibility
4. **Decision:** Aesthetic has final call with UX veto power
